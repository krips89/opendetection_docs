17 ##Basic principles {#idea_list_gsoc201611}
18 The main philosophy of this library is to provide user detection/recognition algorithms in a **common, intuitive and user-friendly APIs** (perhaps we want to build up a structure similar to VTK for object detection algorithms). This is the only reason we have this separate library. Therefore, please go through the existing <a href="inherits.html"><b>classes</b></a> to understand the framework before your application.
19 
20  Thus, all the algorithms/methods you propose in your application have to extend the previous interfaces which will abstract all the implementation details from the user, and at the same time provide users the ability to configure with parameters. In other words, we don't want one of the many demo applications of detection algorithms. We want the detection algorithms under the existing APIs (Trainers and Detectors). This should be the main focus of the contribution.
21 
22 If you think that some of our APIs and framework interfaces are not optimal (in terms of usability and speed or any other reason) and have some suggestions for the improvement, please provide them in your application (specifically in Project Idea 2) as well.

28 ###1. CNN based algorithms in OpenDetection {#idea_list_gsoc20163}
29 
30 **Key tasks**:
31 * Make a short report of the popular and state-of-the-art detection/recognition techniques (to let us know that you have the proficiency in this area)
32 * Add CNN based algorithms in the library with OpenDetection Compatible APIs in the following order: -
33 * Implement OD interface for training and testing (under od::ODTrainer s) neural network architectures with convolutional layer and possible recurrent layers using Caffe.
34 * Implement interface for forward phase of object detection (under od::ODDetector s or one of its subclasses) on some pre-trained caffe models (or trained models from the previous step) based on architectures which have been used for object detection.
35 * Implement train/test data preparation/region extraction and other preprocessing steps (maybe in a separate module).
36 * Implement any other state-of-the-art detection architectures which uses CNNs.
37 
38 **Prerequisites**:
39 
40 * We are expecting a Master student/PhD student in Machine Learning/Computer Vision for this project. If not, you should have a good knowledge in neural networks and experience in training and testing different types of neural networks/CNNs.
41 * Knowledge of Caffe.
42 * High command in C++.
43 
44 **Mentors**: Kripasindhu Sarkar, Aditya Tewari

46 ###2. Framework design and library maintenance {#idea_list_gsoc20164}
47 
48 This is one of the important project and have a very strong influence to the future of the library.
49 
50 **Key tasks**:
51  * Improve the existing OpenDetection framework in terms of usability.
52  * Adding more helper functions/classes to improve the library
53  - Some wrappers for visualization - right now we directly use opencv functions and pcl visualizers
54  - Other changes
55  * Preparing ProjectConfig.cmake file to make the library usable by other projects.
56  * Verifying and fixing cross platform compilation.
57  * Package creation for different platforms.
58  * Reduce dependencies of the library to minimum.
59  * General maintenance
60 
61 **Prerequisites**:

63 * High command in C++ and **CMake** (Good knowledge of CMake is very important).
64 * Knowledge of OpenCV/PCL.
65 * It will be nice if you have already maintained/managed a C++ library before. If not, you should have experience coding in different platforms (at least have knowledge in build toolchains in windows and linux).

69 ###3. Integration of some other popular detection algorithms in OpenDetection {#idea_list_gsoc20165}
70 
71 **Key tasks**:
72 * This is an open idea. We always would like to integrate **state-of-the-art** algorithms in our framework. It would be easy to port an available C++ code in this framework instead of implementing an idea/paper from scratch. So, we expect you to *do some literature review and report different algorithms* with their availability and the ones you would like to add. Please note that, just because the implementation of some algorithm is available online does not mean that it is the state-of-the-art or one of the popular algorithms.
73 * Implement/port the identified detection algorithms under od::ODDetector2D/od::ODDetector3D and the respective *Trainer* s.
74 
75 **Prerequisites**: Expertise in Computer Vision and C++.
76 
77 **Mentors**: Kripasindhu Sarkar, Marco A. Gutiérrez
