5 Detection 2D {#detection_2d1}
6 ===
7 
8 This article goes through the 2D detection methods covered in OD. Specifically, it covers the classes - od::g2d::ODHOGDetector through a tutorial.
9 
10 2D detection methods are performed by the classes Detector2D. They accept a `SceneImage` and performs detection/recognition on them. Currently Detector2Ds are classified into g2d and l2d namespaces. g2d covers detection methods which uses global 2D features (like HOG/Cascade) while l2d covers detection methods which uses local 2D features (like SIFT/SURF/ORB) for detection/recognition. Different 2D detectors that are available currently: od::g2d::ODHOGDetector, od::g2d::ODCascadeDetector, od::g2d::ODFaceRecognizer, od::l2d::ODCADRecognizer2DLocal
11 
12 ##HOG feature based detection {#detection_2d2}
13 
14 HOGDetector is a HOG feature based linear classifier. It accepts an image (od::ODSceneImage), computes its HOG feature (in a multiscale mannar for detectOmni() and a single descriptor from the resized image for detect()), runs a linear SVM obtained either by HOGTrainer through training or some default ones (from OpenCV), and informs if the classifier is true thereby providing detection.
15 
16 A complete example including training is provided in `examples/objectdetector/od_hog_train.cpp`. For positive and negetive example get the data from the INRIA human dataset: http://pascal.inrialpes.fr/data/human/.
17

40 int main(int argc, char *argv[])
41 {
42  std::string trained_data_dir(argv[1]), input_video(argv[2]), output_video = "output.avi";
43  if (argc > 3) output_video = argv[3];
44 
45 
46  //get 3 detectors of different types
47  vector<string> messages; messages.push_back("Original");
48  vector<g2d::ODHOGDetector*> detectors;
49  g2d::ODHOGDetector *detector1 = new g2d::ODHOGDetector; //

53  messages.push_back("OpenCV Daimler People"); detectors.push_back(detector2);
54 
55  g2d::ODHOGDetector *detector3 = new g2d::ODHOGDetector(trained_data_dir);
56  messages.push_back("Custom HOG from trained data"); detectors.push_back(detector3);
57 
58  //init all detectors
59  for (int i = 0; i < detectors.size(); i++) detectors[i]->init();
60 
61  //get scenes
62  od::ODFrameGenerator<od::ODSceneImage, od::GENERATOR_TYPE_DEVICE> frameGenerator(input_video);
63  cv::VideoWriter videoWriter(output_video, CV_FOURCC('M','J','P','G'), 25, sizesingle * 2, true);
64 
65 
66  //GUI
67  cv::namedWindow("Overlay", cv::WINDOW_NORMAL);
68  while(frameGenerator.isValid() && cv::waitKey(33) != 27)
69  {
70  od::ODSceneImage * scene = frameGenerator.getNextFrame();
71 
72  vector<cv::Mat> images_to_show;
73  images_to_show.push_back(scene->getCVImage()); //push the first image
74 
75  //detect 3 times
76  for (int i = 0; i < detectors.size(); i++)
77  {

79  if(detections->size() > 0)
80  images_to_show.push_back(detections->renderMetainfo(*scene).getCVImage());
81  else images_to_show.push_back(scene->getCVImage());
82  }
83

85  cv::imshow("Overlay", multiimage);
86  videoWriter << multiimage;
87 
88  delete scene;
89  }

92 }

You can either train as in `examples/objectdetector/od_hog_train.cpp`, or get the OD pre-trained data from the \ref getting_started2 "Data Repository".

113 ###Code explanation {#detection_2d4}
114 We first init 3 different instances of HOGDetector of different settings.
115 
116  vector<g2d::ODHOGDetector*> detectors;
117  g2d::ODHOGDetector *detector1 = new g2d::ODHOGDetector; //
118  messages.push_back("OpenCV Default People"); detectors.push_back(detector1);
119 
120  g2d::ODHOGDetector *detector2 = new g2d::ODHOGDetector; detector2->setSvmtype(g2d::ODHOGDetector::OD_DAIMLER_PEOPLE);
121  messages.push_back("OpenCV Daimler People"); detectors.push_back(detector2);
122 
123  g2d::ODHOGDetector *detector3 = new g2d::ODHOGDetector(trained_data_dir);
124  messages.push_back("Custom HOG from trained data"); detectors.push_back(detector3);
125 
126 You can set different types of linear SVMs for HOG detector using `setSvmtype` function. For adding a custom SVM use the type OD_CUSTOM and set the linear SVM weight vector using `setSVMDetector()`. You have to then update the other HOG detector parameters accordingly (like winSize etc) with which the your SVM was trained. If you give a trained data directory, it will use the xml file from the directory. Here, we set three different available HOG detectors.
127 
128 After setting all the parameters you need to call `init()` which is done in the following line.
129 
130  //init all detectors
131  for (int i = 0; i < detectors.size(); i++) detectors[i]->init();
132 
133 Then we create a FrameGenerator object, which is a templated class for grabbing both Images and Point Clouds (from kinect). Use od::GENERATOR_TYPE_DEVICE for grabbing frames from camera (kinekt for Point Cloud) or video which is done in the following line.
134 
135  //get scenes
136  od::ODFrameGenerator<od::ODSceneImage, od::GENERATOR_TYPE_DEVICE> frameGenerator(input_video);
137 
138 The next valid frame can be accessed now by `frameGenerator.getNextFrame();` which is being done in loop until exhaustion.
139 
140 Each ImageScene is then checked by the detector using the detectOmni() function which searches the whole image for detections. The resultant Detections2D contains information about the detections made (like Type/Bounding box etc). We use an function `renderMetainfo` to draw the bounding box of all the detections made.
141 
142  //detect 3 times
143  for (int i = 0; i < detectors.size(); i++)
144  {
145  ODDetections2D *detections = detectors[i]->detectOmni(scene);
146  if(detections->size() > 0)
147  images_to_show.push_back(detections->renderMetainfo(*scene).getCVImage());
148  else images_to_show.push_back(scene->getCVImage());
149  }
150 
151 The function `od::makeCanvasMultiImages` take N images and concatenate them in order forming a big image with messages. That concatenated image with having information from all the detectors are then being shown.
152 
153  cv::Mat multiimage = makeCanvasMultiImages(images_to_show, sizesingle, messages);
154  cv::imshow("Overlay", multiimage);
