14 ##Basic principles {#idea_list_gsoc201711}
15 The main philosophy of this library is to provide user detection/recognition algorithms in a **common, intuitive and user-friendly APIs** (perhaps we want to build up a structure similar to VTK for object detection algorithms). This is the only reason we have this separate library. Therefore, please go through the existing <a href="inherits.html"><b>classes</b></a> to understand the framework before your application.
16 
17  Thus, all the algorithms/methods you propose in your application have to extend the previous interfaces which will abstract all the implementation details from the user, and at the same time provide users the ability to configure with parameters. In other words, we don't want one of the many demo applications of detection algorithms. We want the detection algorithms under the existing APIs (Trainers and Detectors). This should be the main focus of the contribution.

20 ##Project Idea summary {#idea_list_gsoc20172}
21 Following are the list of ideas we will focus for this time in decreasing order of priority.

25 ###Improving the Framework for CNNs for 2D detection
26 
27 **Key tasks**:
28 * Make a short report of the popular and state-of-the-art detection/recognition techniques (to let us know that you have the proficiency in this area)
29 * Add CNN based algorithms in the library with OpenDetection Compatible APIs in the following order: -
30 * Implement OD interface for training and testing (under od::ODTrainer s) neural network architectures with convolutional layer and possible recurrent layers using Caffe.
31 * Add to OpenDitection the following algorithms:
32 .. 1. Classification - Types: different options - AlexNet, VGG, Resnet50, Resnet150, GoogLenet.
33 .. 2. Detection - Faster RCNN
34 .. 3. Location probability map
35 
36 **Skills needed**: Good background in Computer Vision research and CNN. We are expecting a graduate student with computer vision/machine learning as his/her research topic.
37 
38 **Mentors**: Kripasindhu Sarkar, Aditya Tewari

41 ###Standardizing all state of the art detection algorithms.
42 How do we bring make the state of the art to a common platform? This projects aims to bring everything to a common interface. This will enable researcher to evaluate their algorithms easily. Initial tasks are:
43 * Identify popular publically available datasets.
44 * Provide easy APIs for classification/detection in 2D and 3D to evaluate on the respective datasets of a new algorithm.

53 ###Python binding
54 
55 Current opendetection implementation is done on C++, however lots of people use python, specially when it comes to rapid prototyping. Having python bindings in our library will help spread the use of the library and expand the options it offers to the users.
56 
57 **Skills needed**: Python, C++, CMake
58 
59 **Mentors**: Marco A. Gutiérrez, Kripasindhu Sarkar
60 
61 ##Development of examples and tutorials
62 Learn by example is a very big and important issue when users come to a new technology. We believe that a nice set of examples and tutorials can help make the user base of OpenDetection grow. They should be easy to follow and test. This can also help get developers interested in collaborating with our community.
63 Skills needed: C++, Markup languages, CMake
64 
65 **Mentors**: Marco A. Gutiérrez, Kripasindhu Sarkar
66 
67 ##Ease library deployment, automated dependencies installation.
68 As a high level development robotics oriented computer vision library it stands on top of other software. Reducing this dependencies to a maximum and easing installation procedures, like fixing dependencies can also help overcome the initial problems a future user might have when starting to use opendetection. This also includes the development of different operating systems packages for easy installation.
69 
70 **Skills needed**: CMake, package management knowledge, C++
71 
72 **Mentors**: Marco A. Gutiérrez, Kripasindhu Sarkar
