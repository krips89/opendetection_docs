Google Summer of Code 2016 - Ideas

This year we plan to participate in Google Summer of Code as an organisation. Student participating should already have some idea of the detection framework of the library. The best way to get started is to follow some examples and demos after going through Getting Started guide.

Application should be done early and go through the mentors for few iterations. Post the application at https://groups.google.com/forum/#!forum/opendetection as well as the respective mentors.

Project Idea summary

Following are the list of ideas we will focus for this time in decreasing order of priority.

1. CNN based algorithms in OpenDetection

Key tasks:

  • Make a short report of the popular and state-of-the-art detection/recognition techniques (to let us know that you have the proficiency in this area)
  • Add CNN based algorithms in the library with OpenDetection Compatible APIs (probably using Caffe).
    • Having a simple CNN based model using Caffe is a simple task. Wrapping this under API should be the first step.
    • The next steps should be to make it more configurable and adding more techniques (different models/preprocessing techniques/architectures etc.) under different Detectors.

Prerequisites:

  • We are expecting a Master student/PhD student in Machine Learning/Computer Vision for this project. If not, you should have a good knowledge in neural networks and experience in training and testing different types of neural networks/CNNs.
  • Knowledge of Caffe is plus.
  • High command in C++.

Possible mentors: Kripasindhu Sarkar, Aditya Tewari

2. Maintenance and expansion to different platforms

Key tasks:

  • Verifying and fixing cross platform compilation.
  • Package creation for different platforms.
  • Reduce dependencies of the library to minimum if possible.
  • Preparing ProjectConfig.cmake file to make the library usable by other projects.
  • General maintenance, bug fixing, etc.
  • Adding more helper functions/classes to improve the library.

Prerequisites:

  • We will prefer someone who has already maintained/managed a C++ library before. If not, you should have experience coding in different platforms and bug fixing.
  • High command in C++ and CMake (Good knowledge of CMake is very important).
  • Good knowledge of Software Engineering principles.
  • Knowledge of OpenCV/PCL is plus.

Possible mentors: Marco A. Gutiérrez, Kripasindhu Sarkar

3. Integration of some other popular detection algorithms in OpenDetection

Key tasks: This is an open idea. We always would like to integrate state-of-the art algorithms in our framework. It would be easy to port an available C++ code in this framework instead of implementing an idea/paper from scratch. So, we expect you to do some literature review and report different algorithms with their availability and the ones you would like to add.

Prerequisites: Expertise in Computer Vision and C++.

Possible mentors: Kripasindhu Sarkar, Marco A. Gutiérrez
